# Project: Digital Bedroom Homepage

## Overview

Redesign the root homepage of my existing personal website into an immersive interactive experience while preserving the existing application.

This should not become another developer portfolio.

It should become a digital place.

The homepage represents my own cosy room on the internet. Visitors should feel as though they have entered someone's personal space that has existed for years and accumulated personality over time.

The room serves as the gateway into the rest of the website.

---

# Existing Application

## Framework

* Next.js (latest App Router)
* TypeScript
* Existing application already deployed on Vercel
* Single GitHub repository
* Existing pages must remain functional
* Existing routes should not be broken

Do **not** create multiple applications or a monorepo.

---

# Core Philosophy

Avoid making another portfolio homepage.

Instead, build a memorable experience.

The room should communicate personality before presenting information.

Users should spend time exploring before clicking "Enter Website."

Think of it as a digital home rather than a landing page.

---

# Emotional Goals

The homepage should feel:

* warm
* nostalgic
* calm
* handcrafted
* playful
* slightly mysterious
* rewarding to explore
* polished
* timeless

Avoid:

* corporate aesthetics
* SaaS landing pages
* minimalism for its own sake
* excessive text
* overwhelming animations
* sensory overload

# Visual Style

Primary inspirations:

* early 2000s personal websites
* Y2K nostalgia
* pixel art

---

# Accessibility

Meet WCAG AA where possible.

Include:

* keyboard navigation
* screen reader support
* reduced motion mode
* captions where appropriate
* colour contrast compliance

---

# Performance

Aim for:

* Lighthouse 95+
* responsive loading
* code splitting
* lazy loading
* image optimisation
* efficient animation loops

Do not sacrifice responsiveness for visual effects.

---

# Development Process

Do not stop after the first working implementation.

Instead:

1. Build an initial version.
2. Review visual quality.
3. Review UX.
4. Review accessibility.
5. Review animation quality.
6. Review performance.
7. Refactor where needed.
8. Improve interactions.
9. Polish transitions.
10. Repeat until no obvious improvements remain.

Assume you are shipping a portfolio piece worthy of awards rather than completing a task.

---

# Definition of Success

The homepage should make visitors pause and explore.

Within 30 seconds they should think:

"I've never seen a personal website quite like this."

It should feel less like opening a webpage and more like stepping into someone's digital room, with every interaction reinforcing the illusion of a living, evolving personal space that invites exploration before seamlessly leading into the rest of the website.
